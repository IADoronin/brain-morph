# ТЗ: napari-фронтенд для Brain Morph Pipeline

## 1. Назначение

Десктопное приложение на базе **napari** для 3D-регистрации изображений мозга.
Целевая аудитория — биологи и врачи без навыков программирования.

Пользователь:
1. загружает два 3D-объёма (moving + fixed),
2. при необходимости рисует маску ткани,
3. задаёт параметры pipeline в GUI-панели,
4. запускает регистрацию и наблюдает за прогрессом в реальном времени,
5. получает выровненный объём в виде нового слоя napari и сохраняет результат.

### Режимы подключения к бэкенду

| Режим | Запуск бэкенда | URL |
|-------|---------------|-----|
| **Local** | napari сам запускает `uvicorn` как subprocess на localhost | `http://127.0.0.1:8000` |
| **Remote** | бэкенд уже запущен на лаб-сервере | пользователь вводит URL вручную |
| **Colab** | FastAPI + ngrok в Colab, пользователь вставляет публичный URL | вставляется вручную |

napari **не** запускается в Docker. FastAPI-бэкенд опционально может быть в контейнере на сервере.

## 3. Функциональные требования

### 3.1 Загрузка данных

- Загрузка **moving** и **fixed** изображений из файлов `.nii`, `.nii.gz` (через `Volume.load_nii`).
- Поддержка TIFF-серий — в MVP **не требуется** (заглушка).
- После загрузки каждый объём добавляется в napari как слой `Image` с именем `moving` / `fixed`.
- Если файл многоканальный `(C, D, H, W)`, C > 1 — отображается первый канал, остальные доступны через ползунок napari.

### 3.2 Маска ткани

- Кнопка **"Создать маску"** добавляет слой `Labels` поверх `moving` с именем `mask`.
- Пользователь рисует маску стандартными инструментами napari (кисть, заливка).
- Маска передаётся в бэкенд при запуске регистрации.
- Если маска не создана — регистрация запускается без маски.

### 3.3 Настройка параметров pipeline

Панель **PipelinePanel** содержит:

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| Backend URL | str | `http://127.0.0.1:8000` | URL FastAPI бэкенда |
| Backend mode | выбор | `local` | `local` / `remote` / `colab` |
| Стадии | таблица | 4 строки | grid\_shape, n\_steps, lam, image\_scale |
| Optimizer | выбор | `SA` | `SA` / `Gradient` / `Hybrid` |
| λ (lam) | float | `1e-3` | Вес регуляризации |
| Similarity | выбор | `corr` | `corr` / `ncc` / `mse` |
| Channel weights | str | пусто | Веса каналов через запятую, напр. `0.5,0.3,0.2` |
| callback\_freq | int | `50` | Каждые N шагов SA отправляет прогресс |

### 3.4 Запуск и остановка

- Кнопка **"Регистрация"** — отправляет POST-запрос на бэкенд, запускает SSE-поток прогресса.
- Кнопка **"Стоп"** — отправляет DELETE/POST `/stop` на бэкенд, прекращает регистрацию.
- Во время регистрации кнопка "Регистрация" заблокирована.

### 3.5 Прогресс в реальном времени

Бэкенд транслирует события через **Server-Sent Events** (SSE).

Каждое событие — JSON:
```json
{
  "stage": 2,
  "step": 150,
  "cost": -0.423,
  "warped_slice": "<base64 PNG среза>"
}
```

Фронтенд при получении события:
1. Обновляет **cost plot** (matplotlib или pyqtgraph в отдельном виджете): ось X — шаги, ось Y — cost.
2. Обновляет **warped preview** — слой `Image` с именем `warped_preview` в napari (центральный аксиальный срез).
3. Обновляет статусную строку: `Стадия 2/4 | Шаг 150 | Cost: -0.423`.

### 3.6 Результат

После завершения регистрации:
- Получить финальный варпированный объём с бэкенда (`GET /result`).
- Добавить три слоя в napari:
  - `warped` — трансформированный moving
  - `fixed` — уже есть, не дублировать
  - `diff` — `|warped - fixed|` (для визуальной проверки качества)
- Показать диалог: "Регистрация завершена. Сохранить результат?"

### 3.7 Сохранение результатов

- Кнопка **"Сохранить"** в панели результатов.
- Диалог выбора файла (`.nii.gz` по умолчанию).
- Сохраняет слой `warped` через `Volume.save_nii(path)`.
- Опционально — сохранить сетку деформаций (`grid_result.pt`) для повторного использования.

## 5. HTTP-клиент (`src/ui/client.py`)

Все запросы к бэкенду инкапсулированы в классе `BrainMorphClient`.

```python
class BrainMorphClient:
    def __init__(self, base_url: str): ...

    def check_connection(self) -> bool:
        """GET /health → True/False"""

    def start_registration(
        self,
        moving_path: str,
        fixed_path: str,
        stages: list[dict],
        mask_array: np.ndarray | None,
        channel_weights: list[float] | None,
        optimizer: str,
        similarity: str,
        callback_freq: int,
    ) -> str:
        """POST /register → job_id"""

    def stream_progress(
        self,
        job_id: str,
        on_event: Callable[[dict], None],
    ) -> None:
        """GET /progress/{job_id} — SSE, вызывает on_event для каждого события"""

    def stop(self, job_id: str) -> None:
        """POST /stop/{job_id}"""

    def get_result(self, job_id: str) -> np.ndarray:
        """GET /result/{job_id} → warped volume (C, D, H, W) numpy"""
```

Используется **httpx** (sync клиент для простоты; SSE-поток — в отдельном потоке через `threading.Thread`).

## 8. MVP-ограничения

| Функция | MVP | Будущее |
|---------|-----|---------|
| Форматы входных данных | только NIfTI (`.nii`, `.nii.gz`) | TIFF-серии |
| Количество пар | одна пара за раз | очередь заданий |
| Предобработка | нет (LoG, нормализация — вручную перед загрузкой) | встроенный препроцессинг |
| Offline-режим | нет (только через HTTP к бэкенду) | встроенный local backend |
| Редактор стадий | фиксированная таблица из 4 строк | динамическое добавление/удаление стадий |
| Просмотр прогресса | один аксиальный срез в napari | три проекции (axial/coronal/sagittal) |
| Отмена/пауза | только "Стоп" (прерывает полностью) | пауза с возможностью продолжить |

## 9. Сценарий использования (happy path)

1. Пользователь открывает napari → виджеты появляются автоматически (плагин).
2. В **LoadPanel**: нажимает "Обзор" для moving и fixed, нажимает "Загрузить" → оба объёма появляются в viewer.
3. Опционально: нажимает "Создать слой Labels", рисует маску.
4. В **PipelinePanel**: выбирает режим `local`, оставляет параметры по умолчанию.
5. Нажимает "Регистрация" → бэкенд запускается, начинается SSE-поток.
6. В **ProgressPanel** отображается cost plot, в viewer обновляется `warped_preview`.
7. По завершении: слои `warped` и `diff` появляются в viewer.
8. Нажимает "Сохранить", выбирает путь → файл `.nii.gz` сохранён.

## 10. API бэкенда (контракт для фронтенда)

Фронтенд ожидает следующие эндпоинты на FastAPI:

| Метод | URL | Тело / Параметры | Ответ |
|-------|-----|-----------------|-------|
| GET | `/health` | — | `{"status": "ok"}` |
| POST | `/register` | JSON с параметрами задания | `{"job_id": "..."}` |
| GET | `/progress/{job_id}` | — | SSE-поток событий прогресса |
| POST | `/stop/{job_id}` | — | `{"status": "stopped"}` |
| GET | `/result/{job_id}` | — | numpy array в бинарном формате (`.npy`) или JSON |

Событие SSE (один шаг):
```json
{
  "stage":        2,
  "n_stages":     4,
  "step":         150,
  "n_steps":      2000,
  "cost":         -0.423,
  "warped_slice": "<base64 PNG>"
}
```
